# Weather Comparison Frontend

Static, self-contained frontend for comparing weather data between two cities.
No build step, no Node.js, no server-side code required.

## Local development

Serve the repo root with Python's built-in HTTP server (use the project venv):

```bash
/Users/ghjalmar/Documents/Fondur/GbgRvkVedur/.venv/bin/python -m http.server 8080 --directory .
```

Then open **http://localhost:8080/frontend/** in your browser.

> **Why a local server?** ES module `import` statements are blocked by browsers on
> `file://` URLs. A local HTTP server solves this without any build tooling.

## File structure

```
frontend/
  index.html        Single-page app: two city pickers + chart grid
  styles.css        Minimal responsive styles
  app.js            Main wiring: fetch, cache, render Plotly charts
  aggregations.js   Pure JS port of scripts/aggregations.py
  i18n.js           Translation table (Icelandic + English)
  README.md         This file
```

The app expects a `data/` directory **next to `index.html`** containing:

```
data/
  cities.json
  {slug}.json            (daily records — loaded only when year range != default)
  {slug}_summary.json    (pre-aggregated — used for default 2010-2025 range)
  quality.json
```

## Deploying to GitHub Pages

1. **Regenerate the data** (if the source CSVs have changed):

   ```bash
   /Users/ghjalmar/Documents/Fondur/GbgRvkVedur/.venv/bin/python scripts/main.py
   ```

2. **Copy frontend files** to your Pages repo root (or `docs/`):

   ```bash
   cp -r frontend/* /path/to/pages-repo/
   ```

3. **Copy the data directory** alongside `index.html`:

   ```bash
   cp -r output/data /path/to/pages-repo/data
   ```

   Your Pages repo should end up with:
   ```
   index.html
   styles.css
   app.js
   aggregations.js
   i18n.js
   data/
     cities.json
     reykjavik.json
     reykjavik_summary.json
     ...
   ```

4. **Commit and push** — GitHub Pages will serve automatically.

## Keeping data in sync

The `data/` directory is generated by `scripts/build_site_data.py` (called via
`scripts/main.py`). Whenever source CSVs are updated, re-run that script and
copy `output/data/` over to the Pages repo.
